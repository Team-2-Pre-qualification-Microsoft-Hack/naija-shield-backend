# NaijaShield Backend

**Real-time fraud detection and intelligence infrastructure for Nigerian telecommunications.**

NaijaShield intercepts phone calls and SMS messages at the network level, classifies them with AI, and delivers instant warnings — without requiring an app on the user's device. The backend comprises two cooperating services: a .NET Core API that orchestrates the full threat pipeline and enterprise intelligence layer, and a Python FastAPI sidecar that handles voice transcription and deepfake analysis.

## Overview

| Service | Runtime | Role |
|---|---|---|
| **Core API** | ASP.NET Core (.NET 9, C#) | Threat pipeline, persistence, fraud ring intelligence, authentication, SignalR push |
| **AI Voice Sidecar** | Python 3.10+ (FastAPI) | Speech-to-text (Azure Speech), deepfake probability scoring |

**Live deployment:** `https://api-naijashield-dev-a5ggd0exe2dmccf2.eastus-01.azurewebsites.net`

## Key Features

**Threat Detection**
- Real-time classification of SMS and voice calls using Azure OpenAI (GPT-5.4 mini) via Microsoft Semantic Kernel
- Supports five Nigerian languages: English, Pidgin, Yoruba, Hausa, and Igbo — including code-switching
- Covers the major attack typologies: OTP phishing, bank impersonation, USSD tricks, vishing, and family social engineering
- Risk scoring (0–100) with deterministic status thresholds: Blocked ≥ 85, Monitoring ≥ 50, Allowed < 50

**Fraud Ring Intelligence**
- Union-Find connected-component analysis over recent blocked incidents
- Detects coordinated rings by two signals: shared victims (multiple callers targeting the same number) and time-proximity bursts (same classification within a 90-minute window)
- Deterministic ring IDs (SHA-256 of sorted caller numbers) — the same ring retains its ID across analysis runs
- Ring severity scoring factors in caller count, victim count, and recency

**Voice Call Interception**
- Integrates with Jambonz SIP for real-time call forking and Live Call Control (LCC)
- Africa's Talking voice webhooks for programmable call routing
- Synthesises localised audio warnings using Azure Neural TTS (`en-NG-EzinneNeural` — Nigerian English)

**Enterprise Intelligence Layer**
- Geospatial heatmap data: every incident carries lat/lng resolved from the caller's MSISDN
- Compliance-ready regulatory reports for CBN, NCC, EFCC, and NDPC, narrative-generated by Azure OpenAI
- Public scammer reputation API — any bank or fintech can query a number's fraud history without authentication
- Real-time dashboard push via Azure SignalR

**Security and Access Control**
- Invitation-only authentication: no open registration
- Three roles with enforced permissions: `SOC_ANALYST`, `COMPLIANCE_OFFICER`, `SYSTEM_ADMIN`
- JWT access tokens (1-hour) with rotating refresh tokens (7-day), stored server-side in Cosmos DB
- BCrypt password hashing (cost factor 12), per-IP rate limiting, Azure Key Vault for all secrets

## API Reference

All endpoints are served from the Core API. The Python sidecar is an internal dependency — it is not exposed to external consumers.

### Authentication

| Method | Endpoint | Auth | Description |
|---|---|---|---|
| `POST` | `/api/auth/login` | — | Email and password login; returns access + refresh tokens |
| `POST` | `/api/auth/invite` | `SYSTEM_ADMIN` | Create an invitation for a new user (48-hour expiry) |
| `POST` | `/api/auth/invite/accept` | — | Accept an invitation and set a password |
| `POST` | `/api/auth/refresh` | — | Exchange a refresh token for a new token pair |
| `POST` | `/api/auth/logout` | Bearer | Invalidate the current refresh token server-side |
| `GET` | `/api/auth/me` | Bearer | Return the authenticated user's profile |
| `GET` | `/api/auth/users` | `SYSTEM_ADMIN` | List all users in the system |

### Threat Incidents

| Method | Endpoint | Auth | Description |
|---|---|---|---|
| `GET` | `/api/incidents` | — | Paginated threat feed (`?limit=50`) |
| `GET` | `/api/incidents/stats` | — | KPI summary: totals by status, channel, and state |
| `GET` | `/api/incidents/heatmap` | — | Lat/lng points for Azure Maps heatmap rendering |
| `POST` | `/api/incidents/seed` | Bearer | Insert realistic demo incidents (`?count=40`) |

### Fraud Ring Intelligence

| Method | Endpoint | Auth | Description |
|---|---|---|---|
| `GET` | `/api/fraud-rings` | — | All rings detected in the last N hours (`?hours=72&minCallers=2`) |
| `GET` | `/api/fraud-rings/{ringId}` | — | Full ring profile including every attributed incident |

Ring objects include: `ringId`, `ringSeverityScore` (0–100), `callerNumbers`, `victimNumbers`, `classifications`, `languages`, `states`, `edgeBasis` (`SharedVictim` / `TimeProximity` / `Mixed`), `isActive`.

### Number Reputation

| Method | Endpoint | Auth | Description |
|---|---|---|---|
| `GET` | `/api/numbers/{phone}/reputation` | — | Fraud reputation of a phone number: score, verdict (`HIGH_RISK` / `SUSPICIOUS` / `LOW_RISK` / `CLEAN` / `UNKNOWN`), incident history |

### Compliance Reports

| Method | Endpoint | Auth | Description |
|---|---|---|---|
| `POST` | `/api/reports` | Bearer | Generate and persist a new AI-written report (`CBN`, `NCC`, `EFCC`, `NDPC`, `GENERAL`) |
| `GET` | `/api/reports` | Bearer | List all saved reports (metadata only) |
| `GET` | `/api/reports/{id}` | Bearer | Retrieve a full report by ID |

### Ingestion Webhooks

| Method | Endpoint | Auth | Description |
|---|---|---|---|
| `POST` | `/api/ingest/sms` | — | Africa's Talking SMS delivery webhook |
| `POST` | `/api/at/voice` | — | Africa's Talking voice call webhook |
| `POST` | `/api/jambonz/webhook` | — | Jambonz SIP session hook (call routing + LCC) |

### Demo and Testing

| Method | Endpoint | Auth | Description |
|---|---|---|---|
| `GET` | `/api/demo/scam-scripts` | — | List the built-in Nigerian scam transcripts (English, Pidgin, Yoruba, Hausa, Igbo) |
| `POST` | `/api/demo/simulate-scam-call` | — | Run the full threat pipeline on any built-in script; returns step-by-step timing |
| `GET` | `/api/demo/voice-warning` | — | Synthesise a Nigerian-English audio warning (`audio/mpeg`) |
| `GET` | `/demo/call` | — | Self-contained browser demo page |
| `GET` | `/api/test-scam` | — | Smoke test: verifies Azure OpenAI and Cosmos DB connectivity |

## Local Development

### Prerequisites

- [.NET 9 SDK](https://dotnet.microsoft.com/download/dotnet/9.0)
- [Python 3.10+](https://www.python.org/downloads/)
- [Azure CLI](https://learn.microsoft.com/en-us/cli/azure/install-azure-cli) authenticated via `az login`
- An `appsettings.Development.json` file in the project root (see **Configuration** below)

### 1. Clone the Repository

```bash
git clone https://github.com/Team-2-Pre-qualification-Microsoft-Hack/naija-shield-backend.git
cd naija-shield-backend
```

### 2. Configure Local Secrets

Create `appsettings.Development.json` in the project root (this file is git-ignored):

```json
{
  "Cosmos-Connection-String": "AccountEndpoint=https://...;AccountKey=...;",
  "OpenAI-Key": "...",
  "Azure-Speech-Key": "...",
  "Azure-Speech-Region": "swedencentral",
  "SignalR-Connection-String": "Endpoint=https://...;AccessKey=...;Version=1.0;",
  "ACS-Email-Connection-String": "endpoint=https://...;accesskey=...;",
  "ACS-Email-Sender": "DoNotReply@<your-acs-domain>.azurecomm.net",
  "AT-Username": "sandbox",
  "AT-API-Key": "...",
  "AzureOpenAI": {
    "Endpoint": "https://<your-resource>.openai.azure.com/",
    "DeploymentName": "gpt-4o"
  },
  "Jwt": {
    "Secret": "at-least-32-character-local-dev-secret",
    "Issuer": "NaijaShield",
    "Audience": "NaijaShieldClient",
    "AccessTokenExpiryMinutes": 60,
    "RefreshTokenExpiryDays": 7
  },
  "Jambonz": {
    "AccountId": "...",
    "ApiBaseUrl": "https://your-jambonz-instance",
    "ApiToken": "...",
    "ServerUrl": "https://your-backend-url"
  }
}
```

### 3. Start the Core API

```bash
dotnet restore
dotnet run
```

The API starts on `http://localhost:5000` / `https://localhost:5001`.  
Swagger UI is available at `http://localhost:5000/swagger`.

### 4. Start the AI Voice Sidecar

```bash
cd sidecar

# Create and activate a virtual environment
python -m venv venv
venv\Scripts\activate          # Windows
# source venv/bin/activate     # macOS / Linux

pip install -r requirements.txt
python main.py
```

The sidecar starts on `http://localhost:8000`. The Core API sends voice analysis requests here automatically when processing call events.

## Deployment

The backend is deployed to Azure App Service (eastus) via GitHub Actions.

### CI/CD Pipeline

Every push to `main` triggers `.github/workflows/deploy.yml`, which:
1. Builds and publishes the .NET project
2. Packages the output as a zip artifact
3. Deploys to `api-naijashield-dev` App Service using the `AZURE_WEBAPP_PUBLISH_PROFILE` repository secret

### Manual Deployment

```powershell
dotnet publish --configuration Release --output ./publish
Compress-Archive -Path .\publish\* -DestinationPath .\publish.zip -Force
az webapp deploy `
  --resource-group rg-naijashield-dev `
  --name api-naijashield-dev `
  --src-path .\publish.zip `
  --type zip
```

### Demo Data Seeding (Azure Logic Apps)

The `azure/` directory contains ARM templates for Logic App workflows that seed realistic demo data:

| Template | Purpose |
|---|---|
| `logicapp-reputation-seeder.json` | Seeds incidents from multiple scammer numbers to populate the reputation API |
| `logicapp-ring-seeder.json` | Seeds 9 incidents across 3 scammer numbers sharing 4 victims, triggering fraud ring detection |

Deploy and trigger via `azure/deploy-reputation-seeder.ps1`.

## User Roles

NaijaShield is an enterprise B2B platform with invitation-only access. There is no open registration.

| Role | Token Value | Access |
|---|---|---|
| SOC Analyst | `SOC_ANALYST` | Threat feed, overview dashboard |
| Compliance Officer | `COMPLIANCE_OFFICER` | Compliance page, report generation |
| System Admin | `SYSTEM_ADMIN` | All pages, user management, invitations, settings |

The default admin account (`admin@naijashield.ng`) is seeded automatically on first startup. Its credentials must be changed before any production deployment.

## Security

| Control | Implementation |
|---|---|
| Password storage | BCrypt, cost factor 12 |
| Access tokens | HS256 JWT, 1-hour expiry |
| Refresh tokens | Server-side (Cosmos DB), 7-day expiry, invalidated on logout |
| Rate limiting | 5 failed login attempts per IP → 15-minute lockout |
| Secret management | Azure Key Vault in production; `appsettings.Development.json` (git-ignored) locally |
| PII protection | PII redaction stage before AI analysis; no BVN/NIN in logs |
| Transport | HTTPS enforced in production (Azure App Service TLS) |
